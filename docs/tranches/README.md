# Tranches

For every *requested set of changes*, we keep a tranche file here that lists the concrete, verifiable items addressed in that pass.

## Naming convention

`YYYY-MM-DD-tranche-NNN-short-title.md`

## Rules

- Every actionable item must be a markdown checkbox.
- Items must be implementable/verifiable by this repo agent (no manual QA checkboxes).
- Completed work must be checked off; partial work must be split.
